Never Open The Door — Case Study

Engineering psychological tension through narrative-state environmental controllers.

01 Project Overview

Never Open The Door explores the intersection of narrative progression and environmental reaction. The project goal was to create a horror experience where tension is a byproduct of discovery rather than timing.

By building a centralized state-discovery system, we ensured that every story fragment found by the player serves as a technical trigger for world distortion.

Design Pillars

  • Atmospheric Depth
  • Narrative Interaction
  • Dynamic Distortion
  • Systemic Pacing

02 Gameplay Loop / Experience Flow

The loop is structured as a feedback system where player investigation drives technical state changes.

01

Active Search: Interaction with environmental props to find "Diary Fragments".

02

State Evaluation: Data collection updates the "Distortion Metadata" in real-time.

03

Event Resolution: Supernatural phenomena priority-queue triggers based on current state.

04 Technical Challenges

State Synchronization

Ensuring environmental changes won't break if player discovers items out of order.

Atmospheric Lighting

Maintaining high performance while using complex volumetric lighting for tension.

Procedural Cues

Synchronizing audio spatialization with visual distortions dynamically.

05 Technical Solutions

We implemented a Discovery Flag Registry that uses bitwise comparison to evaluate narrative progress without heavy string matching.

A custom Environmental Pacing Controller was built to monitor player proximity and discovery rate, adjusting the "Terror Metric" dynamically.

06 Systems Architecture

Narrative State Machine

Centralized engine that manages dependencies between story fragments and world events.

Environmental Mixer

Layered component that blends animations, sound, and shaders based on the "Terror Metric".

07 Key Engineering Takeaways

  • Data-Driven Tension: Pacing should be a result of player data, not fixed timelines.
  • Bitwise Evaluation: For large dependency chains, bitwise flag comparison is the most performant state evaluation.
  • Asset Modularity: Decoupling FX from triggers allows for unlimited scalability in environmental storytelling.

Outcomes

  • Zero-script scalability
  • High player immersion
  • Stable 60 FPS performance
