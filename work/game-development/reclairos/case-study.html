Reclairos — Technical Case Study

Designing scalable combat systems and performance-conscious gameplay architecture for mobile hardware.

Reclairos was developed as a system-driven strategy game where gameplay depth emerges from structured combat systems, enemy behaviors, and encounter design rather than raw difficulty scaling.

The project required balancing gameplay complexity with strict performance constraints while maintaining clarity during large combat encounters.

Project Overview

Reclairos is built around a system-driven combat structure where gameplay depth comes from character roles, enemy behavior, and level progression.

Rather than increasing difficulty through larger enemy health pools or raw damage values, the design focuses on how characters, enemy archetypes, and environments interact during combat encounters.

This approach allowed the game to maintain tactical depth while keeping the core gameplay loop clear and intuitive.

The development focused on building scalable systems instead of isolated features, ensuring the project could expand naturally as more characters, enemies, and encounters were introduced.

Core Gameplay Loop

Players engage in strategic combat encounters where battlefield control is determined through character deployment, enemy counterplay, and evolving encounter compositions.

The core gameplay loop revolves around:

  • Deploying characters with specific combat roles
  • Responding to enemy archetypes and battlefield pressure
  • Managing positioning and timing during combat
  • Adapting strategies as new enemy combinations appear

As the game progresses, new enemy types and encounter compositions introduce additional tactical layers, encouraging players to continuously adapt their strategy.

System Architecture Overview

Reclairos was designed around a modular gameplay architecture where combat encounters emerge from the interaction between several core systems.

Player Strategy

Character classes define the player's tactical tools, while enemy archetypes introduce behavioral challenges that force players to adapt their strategies.

Encounter design combines these elements to create dynamic battlefield situations where positioning, timing, and character roles determine the outcome of combat.

Technical Challenges

Large Character Rosters

Creating separate animation controllers for every unit would be hard to maintain and memory-inefficient. Each unit requires idle, attack, and reaction states.

Combat Readability

Large encounters can become visually chaotic on mobile. Maintaining clarity was essential for strategic responsiveness.

Mobile Constraints

Building visually engaging combat while maintaining stable frame rates under strict rendering and memory limitations.

Animation Pipeline

Instead of building separate animation systems for each unit, the project leverages Unity’s **Animator Override Controllers**.

A single base animation controller manages the core state machine, while individual units replace clips through overrides to preserve underlying logic.

Highlights

  • Shared base animation controller
  • Animator Override Controllers for variation
  • Consistent animation naming conventions
  • Reusable state machine logic

Gameplay Systems Architecture

Designing Combat Around Classes

Characters are structured into functional classes for clear combat roles: frontline durability, ranged pressure, or burst damage.

Enemy Archetype Design

Enemies are built around recognizable behavioral patterns:

  • Aggressive Melee Attackers
  • Heavy Defensive Enemies
  • Fast Disruptive Units
  • Elite Tactical Threats

Performance Engineering

Visuals were designed with a performance-first philosophy rather than relying on heavy post-processing.

  • Reusing assets with controlled visual variation
  • Clear character silhouettes for readability
  • Limiting unnecessary mesh complexity
  • Lightweight particle effects
  • Strict control over active character count
60 FPS Targeted Stability

Level & Encounter Design

Levels support combat pacing and progression. Each stage introduces combinations that build on previously learned mechanics.

  • Gradual introduction of enemy types
  • Controlled difficulty scaling
  • Readable combat arenas
  • Pacing between intense and calm encounters

Narrative Integration

Reclairos integrates a narrative layer that connects gameplay progression with the world. The story provides context for environments and enemy forces, reinforcing world-building through encounters.

Development Philosophy

Our focus was on systems that prioritize clarity, scalability, and performance. Depth emerges from the interaction of roles and behaviors, ensuring stability on mobile without sacrificing gameplay complexity.

Outcome

The architecture developed for Reclairos successfully supports large-scale combat encounters while maintaining stable performance on mobile hardware.

By structuring gameplay around reusable systems rather than isolated features, the project demonstrates how tactical combat games can scale in complexity without sacrificing readability or performance.

The resulting system allows new characters, enemies, and encounters to be introduced efficiently while preserving gameplay clarity and technical stability.

What This Project Demonstrates

Reclairos highlights several key development capabilities:

  • System-driven gameplay architecture
  • Scalable animation pipelines for large character rosters
  • Performance-conscious design for mobile platforms
  • Tactical encounter design built around enemy archetypes
  • Integration of gameplay systems with narrative progression
