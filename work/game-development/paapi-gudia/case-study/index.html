Paapi Gudia — Case Study

Managing high-fidelity atmosphere in standalone VR through optimized architectural pipelines.

01 Project Overview

Paapi Gudia is a cross-platform horror shooter designed to deliver intensive combat and atmospheric tension while maintaining hardware stability on both PCs and Meta Quest headsets.

The project required a "bottom-up" optimization approach where assets were built for the lowest common denominator (standalone VR) while utilizing scalable shaders to enhance visuals on high-end hardware.

Technical Goals

  • Unified VR/PC IO
  • Low Draw Call Count
  • Scalable Atmospherics
  • Dynamic AI Pacing

04 Technical Challenges

VRAM Management

Packing atmospheric textures into tight atlases for Meta Quest memory limits.

Latency Parity

Synchronizing physical interactions across high-latency desktop and low-latency VR.

Lighting Costs

Simulating horror lighting without expensive real-time shadows on mobile chips.

05 Technical Solutions

We developed a Custom URP Lit Shader that includes baked occlusion and fake-specular highlights to simulate depth at near-zero runtime cost.

A Universal Input Wrapper was implemented, mapping VR motion vectors to normalized mouse/gamepad fields for consistent combat feel.

06 Systems Architecture

Cross-Platform I/O

Event-driven input layer that decouples physical controller data from gameplay actions.

Asset Scaler

Automated pipeline that adjusts texture resolution and LOD bias based on the target platform detected at runtime.

07 Key Engineering Takeaways

  • Mobile-First Rendering: Building for the lowest hardware constraints first ensures stable performance on all targets.
  • Unified Abstraction: Input models should be functional, not hardware-dependent, to allow rapid platform porting.
  • Shader-Driven Fidelity: Visual quality on low-end hardware should come from smart shader logic rather than raw texture size.

Outcomes
